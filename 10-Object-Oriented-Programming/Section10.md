# 객체 지향 프로그래밍(OOP)

- 코드 내에서 실생활 개체로 작업을 하는 것

## 객체 vs. 클래스

### 1. 객체

- 코드 안에서 다루는 것

### 2. 클래스

- 객체를 위한 청사진을 정의할 수 있게 해줌
- 재사용 가능한 논리를 한 곳에서 정의한 것을 기반으로 여러 개의 객체를 생성할 때 유용함 => 다중생성가능
- 클래스명은 빈칸이나 대시 부호가 없도록하고, 대문자로 시작하며 구분단어의 시작도 대문자로 표기함
- 클래스는 객체를 대신하지 못함 -> 클래스를 기반으로 객체를 만드는 것이다

### 필드 vs. 프로퍼티

- 클래스 기반으로 객체를 생성할 경우, 필드가 속성이 됨 -> 생성자 함수에서는 그냥 속성이라고 함

### 정적 메서드

- 항상 인스턴스가 이니라 클래스에서 작업하므로 서로 다른 객체에서는 작업하지 않음
